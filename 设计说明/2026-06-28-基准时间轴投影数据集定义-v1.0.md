# 基准时间轴投影数据集定义 v1.0

## 1. 定义

**基准时间轴投影数据集**是基准时间轴页面直接读取的 JSON 数据包。

它不是整部小说的最终结构化数据库，也不是正文内容本身。它的职责是把已经整理出来的叙事对象，投影成基准时间轴页面可以绘制、选择、缩放、拖拽和 Inspector 检查的数据。

标准英文名：

```text
Narrative Timeline Projection Dataset
```

标准内部类型：

```text
narrativeTimelineProjection
```

## 2. 它解决什么问题

基准时间轴页面需要同时显示：

1. 章节覆盖。
2. 故事线轨道。
3. 故事事件条。
4. 人物参与片段。
5. 时间范围和默认视窗。
6. Inspector 需要读取的摘要字段。

这些内容不是纯粹的小说正文，也不是完整的事件对象库。它们是面向“时间轴显示”的投影结果。因此需要单独命名，避免继续混用“story.json”“运行 JSON”“结构化小说数据”等说法。

## 3. 与其他数据层的关系

### 3.1 结构化小说数据

结构化小说数据是更大的上层概念。未来它可以包括：

- 章节对象。
- 事件主记录。
- 人物对象。
- 地点对象。
- 组织对象。
- 故事线定义。
- 事件关系。
- 人物参与关系。
- 章节函数输入输出。

它是全系统的潜在源数据层。

### 3.2 叙事对象库

叙事对象库是结构化小说数据中的核心对象层。它回答：

```text
事件是什么？
人物是谁？
故事线怎么定义？
对象之间有什么关系？
```

### 3.3 基准时间轴投影数据集

基准时间轴投影数据集是页面运行层。它回答：

```text
这些对象在基准时间轴页面上应该怎么显示？
```

所以它可以从叙事对象库生成，也可以在原型阶段由手工样稿或生成脚本生成，但它不应被当成最终事件数据库。

## 4. 当前 JSON 的结构

当前基准时间轴投影数据集至少包含：

| 字段 | 作用 |
| --- | --- |
| `meta` | 数据集元信息。 |
| `timelineConfig` | 默认时间范围。 |
| `chapters` | 章节覆盖切片。 |
| `characters` | 页面需要使用的人物索引。 |
| `tracks` | 故事线轨道。 |
| `events` | 故事事件条和人物参与片段。 |

其中 `events[]` 目前仍是原型期的混合显示数组，包含：

- `kind: "story"`：故事事件条。
- `kind: "character"`：人物参与片段。

后续进入更正式的数据建设时，可以把事件主记录、故事线投影和人物参与片段拆成更明确的对象表；但只要给基准时间轴页面使用，输出层仍应叫“基准时间轴投影数据集”。

## 5. 命名规则

### 5.1 文件名

当前仍沿用 `story-*.json` 文件名前缀，以保持现有页面读取逻辑稳定。

但文档和界面解释中应使用：

```text
基准时间轴投影数据集
```

不要再把这类文件称为：

- `story.json`
- 运行 JSON
- 结构化小说数据
- 时间轴数据

这些说法要么太旧，要么太泛。

### 5.2 meta 字段

新生成的数据集应在 `meta` 中写入：

```json
{
  "datasetType": "narrativeTimelineProjection",
  "projectionTarget": "baselineTimeline",
  "modelVersion": "timeline-projection-v1"
}
```

含义：

| 字段 | 含义 |
| --- | --- |
| `datasetType` | 声明它是叙事时间轴投影数据集。 |
| `projectionTarget` | 声明投影目标是基准时间轴页面。 |
| `modelVersion` | 声明当前投影模型版本。 |

## 6. 页面读取规则

基准时间轴页面读取的是投影数据集清单：

```text
source/data/stories/index.json
```

清单中的每个条目指向一个基准时间轴投影数据集文件。

页面可以切换不同投影数据集，但不应把页面状态写回数据集：

- 缩放比例不写回。
- 横向拖拽偏移不写回。
- 当前选中对象不写回。
- 展开/收起状态暂不写回。
- 真实时间轴 / 章节等宽轴模式暂不写回。

这些是页面视图状态，不是投影数据集本身。

## 7. 生成与维护规则

1. 优先修改源定义、采集样稿或生成脚本，不直接手改投影数据集。
2. 生成脚本输出的 JSON 必须符合 `source/data/schema.json`。
3. 新增投影数据集必须登记到 `source/data/stories/index.json`。
4. 页面显示异常时，先判断问题属于：
   - 源对象定义错误；
   - 投影生成错误；
   - 页面绘制错误；
   - 视图状态错误。
5. 不为“画面好看”改变事件边界和真实时间。

## 8. 当前结论

当前系统此前已经有了数据集读取能力，但没有把数据层正式命名。

从本文件开始，基准时间轴页面读取的这类 JSON 统一称为：

```text
基准时间轴投影数据集
```

它是结构化小说数据面向基准时间轴页面的一种运行投影，而不是结构化小说数据本身。
