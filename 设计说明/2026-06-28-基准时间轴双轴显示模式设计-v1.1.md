# 基准时间轴双轴显示模式设计 v1.1

## 1. 本文目的

当前“基准时间轴原型 V1”已经具备真实时间轴、章节覆盖、故事线、事件块、人物子线、数据集切换和章节函数入口。但它的横向坐标目前只有一种实现：按真实时间比例换算。

用户指出：不同章节的真实时间跨度差异很大。第一章可能跨好几天，后续几章可能集中在同一天。如果只使用真实时间比例，事件排布会极不均匀，导致一些章节和事件在画布上过窄，难以阅读。

因此，本设计补充一个新的页面级能力：

```text
同一份事件数据，在同一个全局时间画布中，支持两种横向投影模式：
1. 标准真实时间轴模式
2. 章节等宽时间轴模式
```

关键要求不是“把页面改成章节轴”，而是建立统一的坐标轴接口。事件、章节、故事线、人物片段都不直接关心当前是哪种模式，只被动接收坐标轴投影结果。

## 2. 当前文档是否完善

结论：当前已有文档不算空白，但还不完善。

已有文档已经覆盖：

1. 核心对象定义：章节、事件、剧情线、人物、人物线、关系、投影。
2. 对象视觉要素：故事线身份色、事件状态局部标记、人物片段虚实。
3. 全局时间轴到章节工作台的路由关系。
4. 事件拆分规则和 C1-C5 数据样稿。

缺口在于：

1. 没有明确“横向坐标轴”本身也是一个页面级投影对象。
2. 没有说明真实时间轴和章节等宽轴如何并存。
3. 没有规定事件块绘制只能消费坐标轴接口，不能直接写死真实时间换算。
4. 基准时间轴 README 中曾经有一条禁止等宽章节轴的旧判断。那条判断在当时是为了防止用简化版替代真实时间底盘；现在应改写为：不能用章节等宽模式替代真实时间模式，但可以把章节等宽模式作为可切换视图存在。

## 3. 设计目标

### 3.1 业务目标

全局时间画布继续承担“看全局故事流”的职责。

用户可以根据任务切换观察方式：

| 模式 | 用途 |
| --- | --- |
| 标准真实时间轴 | 查看事件在真实日期中的距离、密度和跨度。适合检查真实时间逻辑。 |
| 章节等宽时间轴 | 查看每章内部事件布局和跨章推进。适合检查章节节奏、事件归属和故事线发展。 |

两个模式都只改变“横向投影”，不改变事件定义、故事线定义、章节定义。

### 3.2 工程目标

后续编码必须避免到处打补丁。

正确结构是：

```text
数据对象 -> 坐标轴投影 AxisProjection -> 画布绘制
```

错误结构是：

```text
drawEvents 里判断真实时间/章节等宽
drawHeader 里各算一套
drawSelectedChapterColumn 又单独算一套
```

所有需要横向坐标的位置，都应通过同一个轴投影接口获得。

## 4. 非目标

本轮不做以下事情：

1. 不修改事件拆分规则。
2. 不因为章节等宽模式而改写 JSON 中的 `startAt / endAt`。
3. 不删除真实时间轴模式。
4. 不在全局时间线中新增事件因果箭头。
5. 不修改章节函数管理页的业务结构。
6. 不让事件块绘制层直接写分支逻辑处理不同轴模式。

## 5. 核心概念：轴投影对象

坐标轴不是业务对象，它是视图投影对象。

建议在页面内部建立 `AxisProjection` 概念，负责把真实数据映射到画布坐标：

```yaml
AxisProjection:
  mode: realTime | chapterEqual
  worldStart: number
  worldEnd: number
  timeToWorld(time): number
  rangeToWorld(startAt, endAt): { startWorld, endWorld }
  worldToTime(worldX): time
  chapterToWorldRange(chapterId): { startWorld, endWorld }
  getHeaderBands(): HeaderBand[]
  getGridLines(): GridLine[]
```

其中：

- `timeToWorld` 只负责把真实时间投影为抽象横坐标。
- `rangeToWorld` 负责把事件、人物片段、章节范围投影成横向区间。
- `chapterToWorldRange` 负责给章节选区和章节头使用。
- `getHeaderBands` 负责给顶部时间/章节表头使用。
- `getGridLines` 负责给主画布竖向网格使用。

画布绘制层不应该知道内部如何计算真实时间比例或章节等宽比例。

## 6. 模式一：标准真实时间轴

标准真实时间轴是当前默认模式。

### 6.1 坐标规则

```text
worldX = (time - timeline.start) / 一天毫秒数 * dayWidth
```

特点：

1. 真实日期距离越远，画布距离越远。
2. 章节宽度由 `chapter.startAt / chapter.endAt` 决定。
3. 事件宽度由 `event.startAt / event.endAt` 决定。
4. 顶部日期刻度可以按天、周显示。
5. 主画布竖向网格以章节起点为主要分隔。

### 6.2 适用问题

用于检查：

1. 事件是否符合真实时间顺序。
2. 哪些章节真实时间跨度过长或过短。
3. 后台人物片段是否不合理地拖长。
4. 真实日期密度是否支持当前剧情节奏。

## 7. 模式二：章节等宽时间轴

章节等宽时间轴是新增观察模式，不是替代模式。

### 7.1 坐标规则

每个可见章节获得相同宽度：

```text
chapterSlotWidth = 固定章节槽宽
chapterStartWorld = chapterIndex * chapterSlotWidth
chapterEndWorld = chapterStartWorld + chapterSlotWidth
```

事件或人物片段的真实时间仍然有效，但在章节内部按比例投影：

```text
localRatio = (time - chapter.startAt) / (chapter.endAt - chapter.startAt)
worldX = chapterStartWorld + localRatio * chapterSlotWidth
```

如果一个事件跨多个章节：

1. 起点按起始时间所在章节投影。
2. 终点按结束时间所在章节投影。
3. 中间章节自然被覆盖。

如果时间落在两个章节之间的真实时间空隙：

1. 章节等宽模式不额外拉开真实空隙。
2. 时间点吸附到相邻章节边界。
3. Inspector 仍显示真实时间，不因显示模式改变。

### 7.2 顶部表头规则

章节等宽模式下，顶部表头分两层：

| 层级 | 显示内容 | 规则 |
| --- | --- | --- |
| 时间提示层 | 真实日期范围、必要日期点 | 只放在章节槽内部，提示该章真实时间密度。 |
| 章节覆盖层 | C 编号、章节名 | 每章等宽，作为主视觉分隔。 |

主画布竖向网格只使用章节边界，不把每天的日期线贯穿到主画布。

### 7.3 适用问题

用于检查：

1. 每一章内部事件是否看得清。
2. 多条故事线在章节层面的推进是否均衡。
3. 跨章节事件在章节结构里是否过长。
4. 某一章是否缺少主线推进或人物参与。

## 8. 统一绘制规则

### 8.1 事件块

事件块必须从轴投影获取几何信息：

```text
range = axis.rangeToWorld(event.startAt, event.endAt)
x = worldToScreenX(range.startWorld)
width = (range.endWorld - range.startWorld) * scale
```

事件绘制层只处理：

1. 行位置。
2. 最小显示宽度。
3. 颜色和状态标记。
4. 文本裁切。
5. 命中区域。

事件绘制层不负责判断：

1. 当前是真实时间轴还是章节等宽轴。
2. 一个时间点属于第几章。
3. 日期间隔如何压缩或拉伸。

### 8.2 最小宽度

最小宽度规则继续保留。

短事件不能因为缩放而消失，但最小宽度只能影响显示矩形和命中矩形，不能反向修改真实几何终点。

正确原则：

```text
真实几何：由 axis.rangeToWorld 决定
显示矩形：在真实几何基础上套 MIN_EVENT_BLOCK_WIDTH
文本内边距：只影响文字，不影响事件起止坐标
```

这条规则同时适用于真实时间轴和章节等宽轴。

### 8.3 章节选区

选中章节时，背景高亮必须通过：

```text
axis.chapterToWorldRange(chapterId)
```

不能在章节选区里重新用 `timeToWorld(chapter.startAt)` 和 `timeToWorld(chapter.endAt)` 写一套逻辑。

### 8.4 网格线

主画布网格线必须通过：

```text
axis.getGridLines()
```

真实时间轴模式：

- 网格主要使用章节起点。
- 日期线只保留在顶部表头。

章节等宽模式：

- 网格使用章节等宽边界。
- 不把真实日期刻度贯穿到主画布。

## 9. 控件设计

工具栏增加一个分段控件：

```text
轴模式  [真实时间] [章节等宽]
```

默认值：

```text
真实时间
```

切换行为：

1. 不重新加载数据。
2. 不改变事件、章节、故事线对象。
3. 重建当前 `AxisProjection`。
4. 重新计算 `offsetX` 的合法范围。
5. 重新绘制画布。
6. Inspector 当前选中对象可以保留；其几何位置随轴模式更新。

时间范围输入框继续存在。

在真实时间模式下，它决定连续真实时间窗口。

在章节等宽模式下，它决定哪些章节进入“可见章节集合”：只要章节与当前时间范围有交集，就进入等宽排布。

如果当前时间范围没有命中任何章节，页面应回退显示当前基准时间轴投影数据集的全部章节，避免出现空画布。

## 10. 数据结构影响

本轮不要求修改现有故事 JSON 结构。

现有字段继续有效：

```yaml
timelineConfig:
  defaultStart
  defaultEnd
chapters:
  - id
  - title
  - startAt
  - endAt
tracks:
  - id
  - name
  - color
events:
  - id
  - kind
  - track
  - startAt
  - endAt
  - visualState
```

后续可以考虑增加可选字段：

```yaml
timelineView:
  defaultAxisMode: realTime
  chapterSlotWidth: 260
```

但当前不需要把视图状态写入基准时间轴投影数据集。

## 12. 后续编码约束

实现时必须满足：

1. `drawEvents` 不直接计算真实时间差。
2. `drawHeader` 不直接拥有事件坐标逻辑。
3. `drawTracks` 不直接推导日期网格。
4. `drawSelectedChapterColumn` 不单独写章节宽度逻辑。
5. 所有横向投影都走 `AxisProjection`。
6. 最小宽度逻辑只处理显示矩形，不污染真实几何。
7. 真实时间模式必须保持当前视觉和交互基线。
8. 章节等宽模式必须保留缩放、拖拽、选中、Inspector、章节函数入口。

## 13. 验收标准

后续编码完成后，至少要验证：

1. 默认打开仍是标准真实时间轴。
2. 切换到章节等宽后，每个可见章节宽度一致。
3. 同一个事件在两种模式下使用同一份 `startAt / endAt` 数据。
4. 短事件在两种模式下都不会消失。
5. 最小宽度不会让事件终点突破真实投影终点。
6. 选中章节后，选区宽度随模式变化正确。
7. 左侧故事线展开/收起仍能点击，不被事件块穿透干扰。
8. 基准时间轴投影数据集切换后，轴模式仍能重新投影新数据。
9. 章节函数入口仍能带当前章节 ID、标题和真实时间进入工作台。

## 14. 推荐实现顺序

1. 先在代码中建立 `AxisProjection` 构造函数或工厂函数。
2. 把当前真实时间换算迁移为 `realTime` 轴实现。
3. 增加 `chapterEqual` 轴实现。
4. 让 `drawEvents / drawHeader / drawTracks / drawSelectedChapterColumn` 只消费轴接口。
5. 增加轴模式切换控件。
6. 增加脚本检查，禁止事件绘制层重新写死真实时间宽度。
7. 浏览器中验证真实时间模式和章节等宽模式。

## 15. 当前结论

当前页面可以继续演化，但必须先补上“轴投影层”这个概念。

如果直接在现有 `timeToWorld` 上加一堆分支，也能短期跑起来，但会让后续事件块、章节选区、网格、Inspector、拖拽缩放互相污染。正确做法是把“真实时间”和“章节等宽”都做成同一个接口下的两种轴实现，让页面上的对象被动读取投影结果。
