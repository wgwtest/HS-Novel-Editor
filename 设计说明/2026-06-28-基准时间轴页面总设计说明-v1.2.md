# 基准时间轴页面总设计说明 v1.2

## 1. 本文结论

当前页面此前没有一份完整的“页面总设计说明”。

已有文档分别说明了对象定义、事件定义、视觉编码、章节工作台贯通、事件拆分规则和双轴显示模式，但这些文档是按问题逐步补出来的，不等于一份能直接指导 `source/index.html` 后续演化的页面级设计说明。

本文用于补齐这个缺口。它把当前页面作为一个完整产品界面来定义：页面负责什么、管理哪些对象、读取哪些数据、如何绘制、如何交互、哪些内容不能混进来。

## 2. 页面身份

| 项 | 内容 |
| --- | --- |
| 页面名称 | 基准时间轴页面 |
| 当前原型 | 基准时间轴原型 V1 |
| 页面文件 | `原型包/2026-06-22-叙事验证工具-基准时间轴原型-v1/source/index.html` |
| 投影数据集目录 | `source/data/stories/` |
| 投影数据集清单 | `source/data/stories/index.json` |
| 数据结构约束 | `source/data/schema.json` |
| 下钻页面 | `source/chapter-workbench.html` |

页面定位：

```text
基准时间轴页面 = 全局故事流观察、对象选择、章节函数入口。
```

它不是正文编辑器，不是事件详情页，也不是章节工作台。它的职责是让用户在全局范围内看见章节、故事线、事件和人物参与片段如何分布。

## 3. 页面目标

页面要解决四个问题：

1. 全局故事流在哪里推进。
2. 各条故事线在章节和真实时间上的分布如何。
3. 事件、人物片段、章节之间是否有明显的跨度异常或归属异常。
4. 用户选中对象后，能否通过 Inspector 明确判断下一步进入事件、章节还是只做观察。

页面不解决：

1. 不编辑正文。
2. 不自动拆分事件。
3. 不直接编辑事件起因、过程、结果。
4. 不在全局画布中画复杂因果箭头。
5. 不用某一条故事线替代事件定义。

## 4. 页面管理的对象

### 4.1 章节

章节是正文切片，也是章节函数入口。

页面中章节用于：

1. 顶部章节覆盖层。
2. 主画布竖向分隔。
3. 选中后的整段背景高亮。
4. 右侧 Inspector 中的章节摘要。
5. “进入章节函数”按钮的上下文。

章节不是故事线，也不是事件。

### 4.2 故事线

故事线是一级叙事轨道。

页面中故事线用于：

1. 左侧轨道名。
2. 事件块的纵向归属。
3. 同一故事线下事件块主色。
4. 展开/收起人物子线。

故事线的定义依据人物关系、行动关系、利益关系和叙事推进关系，不等于“发生了什么主题”。例如“电视台线”应理解为电视台内部核心利益相关人员之间的关系与处置，而不是抽象的媒体口径处理流程。

### 4.3 事件

事件是叙事变化的最小可追踪对象。

页面中的事件块只是事件在全局时间画布上的投影。它显示：

1. 事件短标题。
2. 事件 ID 或 code。
3. 所属故事线。
4. 时间跨度。
5. 局部状态标记。

事件的完整起因、过程、结果不在本页编辑，应进入事件详情页或后续对象详情页处理。

### 4.4 人物片段

人物片段是“某个人在某条故事线里的参与或后台状态”。

页面中人物片段用于：

1. 展开故事线后显示人物子行。
2. 区分正文可见实段和后台虚段。
3. 解释人物状态延续，但不扩大故事线事件边界。

人物片段不是全局人物传记。

### 4.5 Inspector

Inspector 是选中确认层。

它负责：

1. 显示当前选中对象类型。
2. 显示必要摘要。
3. 显示基础关系。
4. 提供明确进入按钮。

它不承担完整详情编辑。

## 5. 页面布局

页面采用三段结构：

```text
顶部状态栏
主工作区：左侧/中部 Canvas + 右侧 Inspector
```

### 5.1 顶部状态栏

顶部状态栏显示：

1. 页面名称。
2. 当前路径或状态。
3. 当前数据来源。

要求：

- 不放大量说明文字。
- 不占用过多垂直空间。
- 风格应与章节工作台保持一致。

### 5.2 工具栏

工具栏承载页面级控制：

1. 数据集选择。
2. 时间范围起止。
3. 轴模式切换。
4. 缩放控制。
5. 复位、放大、缩小、全展、收起。

工具栏只放操作，不放概念解释。

### 5.3 Canvas 画布

Canvas 是页面主体。

它负责绘制：

1. 时间/章节表头。
2. 左侧故事线与人物子线标签。
3. 主画布行背景。
4. 章节网格。
5. 选中章节背景。
6. 事件块。
7. 人物片段块。
8. 命中区域。

Canvas 不直接保存业务数据。它只读取当前基准时间轴投影数据集和页面状态。

### 7.1 基准时间轴投影数据集

基准时间轴页面读取的 JSON 统一称为“基准时间轴投影数据集”。

它不是最终结构化小说数据库，而是叙事对象面向基准时间轴页面的一种运行投影。它负责把章节、人物、故事线、故事事件和人物参与片段转换成页面可以绘制、选择和 Inspector 检查的结构。

标准定义见：

```text
设计说明/2026-06-28-基准时间轴投影数据集定义-v1.0.md
```

页面代码、schema、数据集清单和后续文档应统一使用这个名称。旧称“运行 JSON”“story.json”“时间轴数据”只能作为历史说明，不再作为当前正式概念。

### 5.4 右侧 Inspector

右侧 Inspector 固定显示当前选中对象。

如果没有选中对象，显示等待选择状态。

选中章节：

- 显示章节 ID、标题、真实时间、输入、内部运作、输出。
- 启用“进入章节函数”。

选中事件：

- 显示事件标题、所属故事线、时间范围、章节范围、输入、输出、风险。
- 启用“拆解事件”或后续“进入事件详情”。

选中故事线：

- 显示故事线名称、人物数、展开状态。
- 点击故事线标签时展开或收起人物子线。

选中人物片段：

- 显示人物、所属故事线、实段/虚段、时间范围。
- 强调人物虚段不能反向拉长事件。

## 6. 数据读取设计

页面读取投影数据集清单：

```text
source/data/stories/index.json
```

清单中每条投影数据集包含：

```yaml
id: 数据集 ID
label: 下拉框显示名
file: 数据文件名
description: 数据说明
```

页面读取具体数据：

```text
source/data/stories/<file>
```

数据结构至少包含：

```yaml
timelineConfig
chapters
characters
tracks
events
```

设计要求：

1. 页面不再把真实故事数据写死在 HTML 里。
2. HTML 内置数据只作为读取失败时的降级演示。
3. 数据集切换不应破坏缩放、拖拽、选中和 Inspector 的基本逻辑。
4. 后续新增故事线分类、事件重拆样稿，都应通过数据集目录进入页面。

## 7. 坐标轴设计

当前页面需要支持两种横向观察模式：

1. 标准真实时间轴。
2. 章节等宽时间轴。

详细方案见：

`2026-06-28-基准时间轴双轴显示模式设计-v1.1.md`

页面级结论：

```text
坐标轴是视图投影对象，不是业务对象。
```

事件、章节、人物片段都应读取统一的轴投影接口，而不是在各自绘制函数中重复计算。

后续实现必须形成类似结构：

```text
data -> AxisProjection -> render
```

不能形成：

```text
drawEvents 一套时间换算
drawHeader 一套时间换算
drawSelectedChapterColumn 一套时间换算
```

## 8. 绘制层级

Canvas 绘制顺序应保持稳定：

1. 页面背景。
2. 顶部时间/章节表头。
3. 轨道行背景。
4. 左侧标签列。
5. 主画布章节网格。
6. 选中章节背景。
7. 事件块和人物片段块。
8. 命中区域记录。

原因：

- 章节选中背景不能盖住事件块。
- 左侧标签列命中优先级必须高于事件块，避免事件穿透导致无法展开故事线。
- 日期刻度不能贯穿主画布并压过章节网格。
- 事件块最小宽度只影响显示，不影响真实几何。

## 9. 交互设计

### 9.1 缩放与拖拽

页面支持：

1. 滚轮缩放。
2. 滑块缩放。
3. 按钮缩放。
4. 横向拖拽平移。

缩放和平移操作只改变视口状态，不改变数据。

### 9.2 对象选中

点击对象只选中，不自动进入详情。

选中后：

1. Canvas 显示选中状态。
2. Inspector 更新当前对象。
3. 进入动作由 Inspector 按钮触发。

### 9.3 故事线展开

点击左侧故事线标签可以展开或收起人物子线。

左侧标签列命中优先级必须高于画布事件块。即使事件块横向覆盖到标签附近，也不能阻止用户展开/收起故事线。

展开/收起是画布视图状态操作，不是进入详情页；它可以立即执行，但仍应同步更新 Inspector，说明当前故事线的展开状态和人物子线数量。

### 9.4 数据集切换

用户通过下拉框切换基准时间轴投影数据集。

切换后：

1. 重新读取投影数据集 JSON。
2. 更新章节、人物、故事线、事件。
3. 重置或校正视口。
4. 清空不再存在的选中对象。
5. 保持页面基本交互可用。

### 9.5 章节函数入口

选中章节后，右侧 Inspector 启用“进入章节函数”。

进入时传递：

```yaml
chapter: 章节 ID
title: 章节标题
startAt: 章节真实开始时间
endAt: 章节真实结束时间
```

章节工作台读取这些参数作为当前章节上下文。

## 10. 视觉编码规则

页面视觉规则以 `2026-06-23-对象可视化要素与颜色编码规则-v1.0.md` 为准。

当前关键规则：

1. 故事线采用身份色优先。
2. 事件状态只做局部标记，不覆盖故事线主色。
3. 人物色只作为人物识别辅助，不决定事件块主色。
4. 人物虚段使用虚线框，不使用实段底边强调线。
5. 章节不使用事件状态色。

## 11. 当前已知技术状态

当前 `index.html` 中仍存在一些页面演进留下的技术债：

1. 横向坐标仍由 `timeToWorld/timeToScreen` 直接按真实时间换算。
2. `drawHeader`、`drawTracks`、`drawEvents`、`drawSelectedChapterColumn` 都直接依赖时间换算。
3. 双轴显示模式尚未编码。
4. 事件最小宽度已有独立逻辑，但后续必须继续保证它不污染真实几何。
5. 页面已有基准时间轴投影数据集读取能力，但投影数据集仍是面向页面显示的轻量结构，不是最终事件数据库。

这些技术状态说明：后续编码应先抽出轴投影层，再加章节等宽模式。

## 13. 后续编码入口

在用户确认本文后，建议后续编码按以下顺序推进：

1. 增加轴模式切换控件的设计实现。
2. 抽出 `AxisProjection`。
3. 把现有真实时间模式迁移到 `realTime` 投影。
4. 增加 `chapterEqual` 投影。
5. 改造 `drawHeader / drawTracks / drawEvents / drawSelectedChapterColumn`。
6. 增加脚本检查，防止事件绘制层重新写死真实时间宽度。
7. 在浏览器中验证两种模式、缩放、拖拽、选中、故事线展开和章节函数入口。

## 14. 页面验收标准

当前页面后续迭代应满足：

1. 用户能看清当前加载的是哪个数据集。
2. 用户能按真实时间查看事件分布。
3. 用户能按章节等宽查看章节内部和跨章推进。
4. 用户能展开/收起故事线人物子线。
5. 用户能选中章节、事件、故事线、人物片段。
6. 点击对象不会自动跳转。
7. Inspector 能解释当前选中对象。
8. 选中章节后能明确进入章节函数。
9. 短事件不会因缩放消失。
10. 事件最小宽度不会导致时间终点错位。
11. 章节等宽模式不会替代真实时间模式。
12. 所有实验继续留在 `验证工具` 内部。

## 15. 当前结论

这个页面需要一份总设计说明，本文就是补齐后的页面级说明。

此前不是完全没有文档，而是文档分散在多个专题里。后续如果继续直接改 `index.html`，很容易又回到“局部补丁越来越多”的状态。现在应以本文作为页面总纲，以双轴设计文档作为下一步编码的直接技术依据。
