# WBS-0 HS-Novel-Editor 研发总纲

## 1. 总目标

将小说叙事验证工具从原型验证资产演进为可长期维护的叙事结构管理工具。项目应支持全局时间轴观察、章节函数分析、事件拆分、故事线分类、人物参与检查、风险报告和后续数据模型产品化。

## 2. WBS 树

| WBS | 名称 | 类型 | 当前状态 | 说明 |
| --- | --- | --- | --- | --- |
| `P0` | 工程底座与治理 | Phase | 进行中 | 建立文档治理、启动入口、计划、验收和协作边界。 |
| `P0.1` | 文档治理底座 | Task | 进行中 | 建立 `DOC/CODEX_DOC/`、事实源索引、WBS、节点合同和验收入口。 |
| `P0.2` | 启动与运行基线 | Task | 待开 | 固定本地服务、端口、启动脚本和页面可用性验证。 |
| `P1` | 叙事对象与数据模型 | Phase | 待开 | 梳理事件、章节、故事线、人物参与、关系和投影层。 |
| `P1.1` | 投影数据集规范收敛 | Task | 待开 | 将当前轻量投影数据集与未来对象库边界分清。 |
| `P1.2` | 事件采集与人物参与精修 | Task | 待开 | 提升 C1-C14 故事线分类和人物参与质量。 |
| `P2` | 基准时间轴页面产品化 | Phase | 待开 | 将当前 V1 原型演进为稳定页面实现。 |
| `P2.1` | 页面运行层整理 | Task | 待开 | 清理 HTML 单文件技术债，保留当前交互能力。 |
| `P2.2` | 双轴与绘制验收 | Task | 待开 | 以浏览器验证真实时间轴和章节等宽轴。 |
| `P3` | 章节函数工作台 | Phase | 待开 | 将章节输入、内部处理和输出事件关系产品化。 |
| `P4` | 报告与同步建议 | Phase | 待开 | 风险报告、章节准备包和同步建议输出。 |
| `P5` | 协作、发布与外部化 | Phase | 待开 | 远端 tracker、发布策略和对外包只在确认后启动。 |

## 3. 当前推进原则

- 当前活动节点是 `P0.1`。
- `P0` 未关闭前，可以做低耦合准备工作，但不应把后续核心实现视为正式启动。
- 人工确认前，节点状态不进入已验收。
- 任何涉及页面外观的后续实现，必须引用已确认原型或当前运行页截图。

## 4. 里程碑

| Gate | 名称 | 关闭条件 |
| --- | --- | --- |
| `G0` | 工程治理入口可用 | 文档根、WBS、节点合同、验收入口、交接规则可被新会话读取。 |
| `G1` | 当前 V1 运行基线可复现 | 本地启动、数据加载、脚本校验和浏览器基本交互有证据。 |
| `G2` | 数据模型边界稳定 | 投影数据集、对象库和正式小说材料边界明确。 |
| `G3` | 基准时间轴页面可产品化 | 页面结构、交互、验收标准和实现拆分方案明确。 |
