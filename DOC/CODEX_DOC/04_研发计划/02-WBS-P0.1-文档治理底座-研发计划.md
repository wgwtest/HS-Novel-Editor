# WBS-P0.1 文档治理底座研发计划

## 1. 工作目标

建立 HS-Novel-Editor 的正式工程文档根，让后续工作不再只依赖散落的设计说明、历史原型和聊天交接。

## 2. 工作内容

1. 创建 `DOC/CODEX_DOC/` 七类文档目录。
2. 写入本地工程策略映射。
3. 写入工程总体分析。
4. 写入设计事实源索引。
5. 写入文档治理与迁移规则。
6. 写入 WBS 总纲和当前节点计划。
7. 写入当前节点合同。
8. 建立验收大纲、验收主入口和过程交接位置。
9. 更新根目录启动与交接文档，使其指向正式文档根。

## 3. 非目标

本节点不做：

1. 不移动历史 `设计说明/`。
2. 不移动历史 `原型包/`。
3. 不重构页面源码。
4. 不创建远端 tracker。
5. 不提交 `验证作品/`。

## 5. 验收方法

1. `rg --files DOC/CODEX_DOC` 能列出正式文档骨架。
2. `git status --short --ignored` 显示 `验证作品/` 仍为忽略项。
3. 基准时间轴现有校验脚本仍通过。
4. 根目录 `CODEX_START_HERE.md` 和 `README.md` 能引导新会话进入 `DOC/CODEX_DOC/`。
