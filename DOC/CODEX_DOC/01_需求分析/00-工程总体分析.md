# 工程总体分析

## 1. 背景

本项目最初是小说结构管理工具的验证工具，用来检查章节、事件、故事线、人物参与和时间轴投影是否能帮助作者理解叙事结构。在原型推进过程中，工具本身的价值超过了单次验证目的，因此需要转为可长期演进的工程项目。

## 2. 当前目标

当前目标不是重写页面，而是先建立工程治理底座：

1. 明确正式文档根。
2. 固定事实源阅读顺序。
3. 把历史设计说明、原型包和当前运行实现分层。
4. 建立 WBS、节点合同、验收入口和会话交接位置。
5. 为后续数据模型、页面实现和产品化迭代提供稳定协作边界。

## 3. 当前系统组成

| 模块 | 当前位置 | 说明 |
| --- | --- | --- |
| 快速启动文档 | `CODEX_START_HERE.md` | 新会话恢复、启动命令、当前主入口。 |
| 当前交接 | `CURRENT_HANDOFF.md` | 历史交接记录，部分内容已落后于源码。 |
| 策划与需求 | `小说叙事验证工具-策划案.md`、`叙事验证工具-需求规格说明-v0.1.md` | 原始产品定位和需求草案。 |
| 专题设计说明 | `设计说明/` | 对象、事件、页面、投影数据集、视觉编码等专题说明。 |
| 当前基准原型 | `原型包/2026-06-22-叙事验证工具-基准时间轴原型-v1/` | 当前运行页面、章节函数页面、数据和脚本。 |
| 历史原型 | `原型包/2026-06-*` | 设计过程记录和截图证据。 |
| 角色头像与草图 | `角色头像/`、`设计草图/` | 视觉资产和草图材料。 |
| 本地素材区 | `验证作品/` | 正文和成熟支撑材料，只读引用，不提交。 |

## 4. 当前已知事实

- 当前页面入口是 `原型包/2026-06-22-叙事验证工具-基准时间轴原型-v1/source/index.html`。
- 当前页面必须通过 HTTP 服务打开，因为要读取本地 JSON 数据。
- 当前页面读取的数据层统一称为“基准时间轴投影数据集”。
- 当前源码已经包含 `createAxisProjection`、真实时间轴和章节等宽轴模式。
- 当前数据脚本已经覆盖投影元信息、人物参与、故事线预览质量、轴模式、短事件宽度、命中优先级和视觉编码检查。

## 5. 当前主要差距

| 差距 | 影响 | 处理方向 |
| --- | --- | --- |
| 正式工程文档根缺失 | 新会话容易只读散落文档，无法稳定判断优先级 | 建立 `DOC/CODEX_DOC/`。 |
| 交接文档状态滞后 | 可能误认为双轴仍未实现 | 新建事实源索引，区分历史交接和当前源码事实。 |
| WBS 与节点合同缺失 | 任务无法稳定拆分、验收和交接 | 建立 WBS 总纲与 `P0.1` 节点合同。 |
| 验收记录无固定位置 | 校验结果容易留在聊天中 | 建立 `06_测试文档/`。 |
| 原型资产多但缺正式索引 | 历史原型和当前基准容易混淆 | 建立原型包索引，保留现有资产根。 |

## 6. 非目标

本轮不做：

1. 不移动 `设计说明/` 和 `原型包/` 中的历史文件。
2. 不修改当前页面行为。
3. 不把 `验证作品/` 纳入 Git。
4. 不创建远端 tracker 或修改 GitHub Project。
5. 不将基准时间轴投影数据集升格为最终小说数据库。
