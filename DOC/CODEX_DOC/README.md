## 协作规则

- 新的稳定工程文档默认写入 `DOC/CODEX_DOC/`。
- 现有 `设计说明/` 作为历史设计事实源保留；`原型包/` 作为历史行为基线和视觉回归基线保留，不作为产品功能主开发目录。
- 数据构建流程文档归入 `DOC/CODEX_DOC/03_规范与流程/01_数据构建流程/`，不继续塞进页面设计说明。
- `验证作品/` 只作为本地素材区，必须保持在 `.gitignore` 中。
- 页面和数据改动必须先确认事实源属于对象定义、投影生成、页面绘制还是视图状态。
- 当前基准页面以 `app/index.html` 为正式实现入口。
