# 文档治理与迁移规则

## 1. 本轮治理原则

本轮采用渐进式对齐，不做大规模搬迁。

原因：

1. 当前主页面、README 和启动文档仍引用 `原型包/` 和 `设计说明/`。
2. 历史原型包包含大量截图和评审记录，直接移动会制造路径断裂。
3. 当前优先级是建立正式治理入口和事实源索引，而不是清理历史。

## 2. 新文档写入规则

| 文档类型 | 写入位置 |
| --- | --- |
| 工程总览、产品边界、模块分析 | `DOC/CODEX_DOC/01_需求分析/` |
| 新稳定设计说明 | `DOC/CODEX_DOC/02_设计说明/` |
| 数据规范、关键流程、设计实现映射 | `DOC/CODEX_DOC/03_规范与流程/` |
| WBS 和阶段计划 | `DOC/CODEX_DOC/04_研发计划/` |
| 单节点执行合同 | `DOC/CODEX_DOC/05_节点合同/` |
| 验收大纲、验收入口、机测、人测、结论 | `DOC/CODEX_DOC/06_测试文档/` |
| 会话交接、历史计划、历史原型评审 | `DOC/CODEX_DOC/07_过程文档/` |
| 原型与附图索引 | `DOC/CODEX_DOC/08_原型与附图/` |

## 3. 旧文档保留规则

- `设计说明/` 暂不搬迁，作为历史专题设计库保留。
- `原型包/` 暂不搬迁，作为当前原型资产根保留。
- 根目录策划案和需求规格暂不搬迁，先由正式文档索引引用。
- 如果后续要迁移旧文档，必须先写迁移计划并列出路径映射。

## 4. 事实源判定顺序

当文档之间出现冲突时，按以下顺序判断：

1. 用户在当前回合的明确指令。
2. 当前源码、数据和可运行页面的可验证事实。
3. `DOC/CODEX_DOC/` 中的正式工程文档。
4. 6 月 28 日面向当前基准时间轴 V1 的设计说明。
5. 旧交接文档。
6. 早期原型和早期设计草案。

## 5. 验证作品边界

`验证作品/` 是本地素材区，包含正文、设定和成熟支撑材料。它不属于本项目默认交付物。

规则：

- 必须保持在 `.gitignore` 中。
- 可以本地只读引用。
- 不直接覆盖其中的正式小说文件。
- 不把其中内容复制到公开仓库文档，除非用户明确要求且确认发布边界。

## 6. 启动文档同步规则

当以下内容变化时，必须同步更新 `CODEX_START_HERE.md` 和本目录 README：

- 当前主页面入口。
- 当前基准原型目录。
- 正式文档根。
- 启动命令。
- 校验命令。
- `验证作品/` 入库边界。
- 当前事实源阅读顺序。
