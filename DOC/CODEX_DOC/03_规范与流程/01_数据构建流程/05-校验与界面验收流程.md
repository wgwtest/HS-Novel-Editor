# 校验与界面验收流程

## 1. 目的

本文定义投影 JSON 生成后如何验证，确保数据能被页面读取，并且不破坏当前基准时间轴功能。

## 2. 最小校验命令

在当前原型 `source` 目录执行：

```powershell
cd "C:\CodexWorkSpace\CodexProject\HS-Novel-Editor\原型包\2026-06-22-叙事验证工具-基准时间轴原型-v1\source"
node scripts/validate-story-datasets.mjs
node scripts/validate-character-participation.mjs
node scripts/validate-storyline-preview-quality.mjs
```

如果只改文档，不改数据，可不运行数据生成脚本，但应检查路径引用和文档入口。

## 3. 完整数据回归命令

当修改源定义、生成脚本或投影 JSON 时，建议运行：

```powershell
node scripts/build-storyline-preview-dataset.mjs
node scripts/validate-story-datasets.mjs
node scripts/validate-character-participation.mjs
node scripts/validate-storyline-preview-quality.mjs
node scripts/validate-axis-modes.mjs
node scripts/validate-event-min-width.mjs
node scripts/validate-hit-priority.mjs
node scripts/validate-visual-encoding.mjs
```

涉及宽屏拖动或视图状态时，再补充：

```powershell
node scripts/validate-view-state-management.mjs
node scripts/validate-wide-canvas-drag.mjs
```

## 4. 页面验收

数据改动完成后，用 HTTP 服务打开页面，不使用 `file://`：

```powershell
cd "C:\CodexWorkSpace\CodexProject\HS-Novel-Editor\原型包\2026-06-22-叙事验证工具-基准时间轴原型-v1\source"
npx --yes vite --host 127.0.0.1 --port 4173
```

验收页面：

```text
http://127.0.0.1:4173/index.html
```

检查项：

1. 下拉框能显示新增数据集。
2. 切换数据集后页面无控制台错误。
3. 章节覆盖层、故事线、事件块和人物子线能绘制。
4. 点击故事事件，Inspector 显示输入、输出、风险。
5. 点击人物片段，Inspector 显示 `action`、`relationChange`、`inspectorSummary`。
6. 故事线展开/收起不受事件块遮挡。
7. 真实时间轴和章节等宽轴都能正常投影。

## 5. 常见失败归因

| 现象 | 优先检查 |
| --- | --- |
| 下拉框没有新数据集 | `data/stories/index.json` 是否登记。 |
| 页面读取失败 | JSON 是否合法，路径是否正确。 |
| 校验缺少 meta | `datasetType`、`projectionTarget`、`modelVersion`。 |
| 人物条显示主事件标题 | `roleBrief` 是否缺失或复制事件标题。 |
| 事件条时间异常 | `startAt` / `endAt` 是否被误改。 |
| 章节等宽模式异常 | 轴投影逻辑是否仍读取统一时间字段。 |

## 6. 提交前检查

提交数据构建相关改动前，至少执行：

```powershell
git status --short
git diff --check
```

如果包含数据或脚本改动，还应附上对应校验命令结果。

如果只包含文档，应说明未运行数据脚本的原因。
