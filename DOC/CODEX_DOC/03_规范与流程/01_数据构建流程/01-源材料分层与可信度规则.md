# 源材料分层与可信度规则

## 1. 目的

本文定义数据构建时如何判断素材可信度，避免把临时推断、候选拆分或运行投影误当成正式小说事实。

## 2. 材料分层

| 层级 | 材料类型 | 当前示例 | 用途 |
| --- | --- | --- | --- |
| L0 | 用户当前明确指令 | 当前会话确认的修改范围 | 本轮最高优先级。 |
| L1 | 正文与成熟设定 | `验证作品/《青云录》/` 下的正文、设定 | 小说事实和语言细节的主要来源，只读引用。 |
| L2 | 分章大纲、简纲、事件脉络 | `致命采访-分章大纲.txt`、`致命采访-事件脉络.md` | 结构采集和事件候选来源。 |
| L3 | 已确认设计规则 | `设计说明/2026-06-24-事件拆分规则与智能体执行准则-v1.0.md` 等 | 判断事件成立、拆分和归属。 |
| L4 | 源定义 / 采集表 | `source/data/storyline-definitions/*.json` | 生成投影 JSON 的结构化输入。 |
| L5 | 运行投影 JSON | `source/data/stories/*.json` | 页面运行数据，不直接代表最终小说数据库。 |
| L6 | 页面视图状态 | 缩放、展开、选中、拖拽位置 | 只属于界面状态，不进入小说事实。 |

## 3. 事实优先级

当材料冲突时，按以下顺序处理：

1. 用户当前明确确认。
2. 正文和成熟设定。
3. 分章大纲、简纲、事件脉络。
4. 已确认设计规则。
5. 源定义或采集表。
6. 运行投影 JSON。
7. 页面显示状态。

运行投影 JSON 不能覆盖正文事实。页面显示异常时，必须先判断问题属于源材料、源定义、生成脚本、schema、页面绘制还是视图状态。

## 4. `验证作品/` 使用边界

`验证作品/` 是本地素材区：

- 可以只读引用。
- 不默认提交。
- 不直接覆盖正式正文。
- 不把正文大段复制进公开工程文档。
- 用于采集时应记录来源文件路径和章节范围，而不是复制全文。

## 5. 候选事实规则

Codex 可以从正文、大纲和事件脉络中提出候选事件，但候选项在用户确认前不得写成正式结构事实。

候选状态建议标记为：

| 状态 | 含义 |
| --- | --- |
| `candidate` | Codex 提出的待审候选。 |
| `needs_review` | 需要用户判断边界或归属。 |
| `confirmed` | 用户已确认，可进入源定义。 |
| `rejected` | 明确不进入本轮数据构建。 |

当前运行投影数据集未完整承载这些状态时，应在采集表或过程记录中保留状态，而不是强行塞进页面 JSON。

## 6. 溯源要求

新增事件、人物参与或故事线归属时，至少记录：

- 来源文件。
- 章节范围。
- 摘要，不复制长段正文。
- 判断依据。
- 是否已人工确认。

如果缺少溯源，默认只能作为候选，不应进入正式投影数据集。
