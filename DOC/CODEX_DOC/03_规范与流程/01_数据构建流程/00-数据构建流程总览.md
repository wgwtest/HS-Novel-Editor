# 数据构建流程总览

## 1. 定位

本文定义 HS-Novel-Editor 中“数据构建流程”的边界和主链路。

数据构建流程不是页面设计，也不是系统对象建模本身。它负责把小说素材、结构判断和人工确认结果整理成基准时间轴页面可读取的投影 JSON。

标准链路：

```text
小说正文 / 大纲 / 事件脉络
-> Codex 采集与拆分候选事件
-> 人工确认事件边界和故事线归属
-> Codex 整理源定义或采集表
-> 脚本生成基准时间轴投影 JSON
-> 校验脚本检查
-> 登记到 stories/index.json
-> 页面读取验收
```

## 2. 与设计文档的边界

| 层级 | 回答的问题 | 事实源 |
| --- | --- | --- |
| 产品 / 页面设计 | 页面消费哪些对象，如何绘制和交互 | `DOC/CODEX_DOC/02_设计说明/00-设计事实源索引.md` |
| 对象模型 | 章节、事件、故事线、人物等对象是什么 | `设计说明/2026-06-22-核心对象定义与系统记录总纲-v0.9.md` |
| 投影数据契约 | 页面运行 JSON 至少有哪些字段 | `设计说明/2026-06-28-基准时间轴投影数据集定义-v1.0.md` |
| 数据构建流程 | 从小说素材到投影 JSON 如何生产、确认、校验 | 本目录 |
| 运行实现 | 页面、脚本和数据文件实际位置 | `原型包/2026-06-22-叙事验证工具-基准时间轴原型-v1/source/` |

数据构建流程可以引用设计文档的规则，但不把采集步骤、脚本运行方式和人工确认状态塞回设计说明。

## 3. 当前事实源

当前可运行数据链路仍以基准时间轴原型 V1 为准：

```text
原型包/2026-06-22-叙事验证工具-基准时间轴原型-v1/source/data/storyline-definitions/
原型包/2026-06-22-叙事验证工具-基准时间轴原型-v1/source/scripts/
原型包/2026-06-22-叙事验证工具-基准时间轴原型-v1/source/data/stories/
```

当前 C1-C14 生成链路：

```text
data/storyline-definitions/storylines-c1-c14-v0.1.json
-> scripts/build-storyline-preview-dataset.mjs
-> data/stories/story-c1-c14-storylines-preview-v0.1.json
-> data/stories/index.json
```

## 4. 核心角色分工

| 角色 | 职责 | 不负责 |
| --- | --- | --- |
| 用户 | 确认事件边界、故事线归属、人物参与口径 | 机械补齐运行 JSON 字段 |
| Codex | 阅读素材、提出候选事件、整理源定义、运行校验 | 擅自把未确认判断写成正式事实 |
| 生成脚本 | 稳定转换源定义为页面运行 JSON | 判断小说情节是否成立 |
| 校验脚本 | 检查字段、人物参与、故事线质量和页面依赖 | 替代人工判断剧情合理性 |
| 页面 | 读取投影 JSON 并展示结构 | 保存视图状态到投影数据集 |

## 5. 文档包结构

| 文件 | 作用 |
| --- | --- |
| `00-数据构建流程总览.md` | 定义总链路、边界和角色分工。 |
| `01-源材料分层与可信度规则.md` | 定义正文、设定、大纲、事件脉络和生成结果的优先级。 |

| `05-校验与界面验收流程.md` | 定义生成后的校验命令和页面验收边界。 |

## 6. 当前非目标

- 不在本目录重新定义页面布局。
- 不在本目录替代对象模型设计。
- 不把 `验证作品/` 正式正文复制进仓库。
- 不把当前投影 JSON 升格为最终结构化小说数据库。
- 不要求所有数据必须由脚本自动判断生成；脚本只负责已确认结构的稳定转换。

## 7. 更新规则

当以下内容变化时，应同步更新本目录：

1. 新增数据源类型。
2. 新增或替换生成脚本。
3. 投影 JSON schema 改变。
4. `stories/index.json` 登记规则改变。
5. 人工确认点或验收命令改变。
6. 从原型 `source/` 迁移到正式 `app/` 后，数据入口发生变化。
