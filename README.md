# 小说叙事验证工具 / HS-Novel-Editor

本仓库是小说叙事结构管理工具的验证与产品化仓库，用于沉淀原型、需求说明、工程文档、验收记录和后续实验代码。当前仓库只管理工具本身，不直接污染外层小说正文和支撑材料。

## 新会话入口

迁移、重建 Git 或新开 Codex 会话后，先读：

```text
CODEX_START_HERE.md
```

长期工程文档入口：

```text
DOC/CODEX_DOC/README.md
```

当前交接说明：

```text
CURRENT_HANDOFF.md
```

## 当前基准

当前基准原型：

```text
原型包/2026-06-22-叙事验证工具-基准时间轴原型-v1/
```

当前主页面：

```text
原型包/2026-06-22-叙事验证工具-基准时间轴原型-v1/source/index.html
```

当前章节函数页面：

```text
原型包/2026-06-22-叙事验证工具-基准时间轴原型-v1/source/chapter-workbench.html
```

这一版作为后续继续迭代的实现基础。它已经表达了以下方向：

- 以时间轴作为对齐底盘。
- 以章节作为基础单元。
- 用多轨事件块表达不同故事线、事件和人物参与条。
- 点击章节或事件只选中，不自动跳转。
- 章节选中后通过“进入章节函数”进入章节工作台。
- 支持基准时间轴投影数据集下拉切换。
- 支持真实时间轴和章节等宽轴两类显示模式。

早期 v6、v7、v9 原型是设计过程记录，不再作为当前实现基准。

## 文档治理规则

- `CODEX_START_HERE.md` 是新会话快速恢复入口。
- `DOC/CODEX_DOC/` 是正式工程文档根，后续稳定计划、合同、验收和交接默认写入这里。
- `设计说明/` 暂作为既有专题设计事实源保留，本轮不搬迁。
- `原型包/` 暂作为版本化原型资产根保留，本轮不搬迁。
- `验证作品/` 是本地素材区，只读引用，必须保持忽略，不能提交。

## 查看原型

先在 `source` 目录启动 HTTP 服务：

```powershell
cd C:\CodexWorkSpace\CodexProject\HS-Novel-Editor\原型包\2026-06-22-叙事验证工具-基准时间轴原型-v1\source
npx --yes vite --host 127.0.0.1 --port 4173
```

再打开：

```text
http://127.0.0.1:4173/index.html
```

不建议直接用 `file://` 打开，因为页面需要读取 JSON 投影数据集。

## 迭代规则

- 新方向或重要评审修正，继续创建新的版本包，不覆盖当前基准。
- 页面读取的 JSON 统一称为“基准时间轴投影数据集”，不是最终结构化小说数据库。
- 页面和数据改动必须先确认事实源属于对象定义、投影生成、页面绘制还是视图状态。
- 外层小说正文、设定、章节文件不属于本仓库的默认改动范围。
